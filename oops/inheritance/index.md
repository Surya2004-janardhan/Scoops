## Inheritance in TypeScript   
Inheritance is a fundamental object-oriented programming concept that allows a new class (called a child or subclass) to inherit properties and behaviors from an existing class (called a parent or superclass). In TypeScript, you can use the `extends` keyword to create a subclass that inherits from a superclass.

## why we use inheritance?
1. **Code Reusability**: Inheritance allows you to reuse code from the parent class, which can save time and reduce redundancy. You can create a new class that inherits the properties and methods of an existing class without having to rewrite the code.    
2. **Extensibility**: Inheritance allows you to extend the functionality of an existing class by adding new properties and methods to the subclass. This makes it easier to create new classes that are based on existing ones, while still maintaining the original functionality.    
3. **Polymorphism**: Inheritance enables polymorphism, which allows you to treat objects of different classes as if they were of the same class. This can simplify code and make it easier to maintain, as you can write code that works with a common interface or base class, rather than having to write separate code for each specific class.    
4. **Improved Organization**: Inheritance can help improve the organization of your code by creating a clear hierarchy of classes. This can make it easier to understand the relationships between different classes and how they interact with each other.    
5. **Encapsulation**: Inheritance can help with encapsulation by allowing you to hide the implementation details of a class while still providing a public interface for other classes to interact with. This can help improve the security and maintainability of your code by preventing other classes from directly accessing the internal workings of a class.    
Overall, inheritance is a powerful tool in object-oriented programming that can help improve code reusability, extensibility, polymorphism, organization, and encapsulation. It allows you to create new classes that are based on existing ones, while still maintaining the original functionality, and can help simplify code and make it easier to maintain.


## simple words explanation of inheritance
Inheritance is like a family tree in programming. Just like how children inherit traits from their parents, in programming, a new class (child) can inherit properties and behaviors from an existing class (parent). This means that the child class can use the code and features of the parent class without having to rewrite it. For example, if you have a class called `Animal` with properties like `name` and `age`, and a method called `makeSound()`, you can create a new class called `Dog` that inherits from `Animal`. The `Dog` class will automatically have the properties and methods of the `Animal` class, and you can also add new properties and methods specific to dogs, like `breed` and `bark()`. This way, you can reuse code and create new classes that are based on existing ones, making your code more efficient and easier to maintain.

## real world example of inheritance
Consider a real-world example of a vehicle management system. We can have a base class called `Vehicle` that contains common properties and methods for all types of vehicles, such as `make`, `model`, `year`, and a method called `start()`. Then, we can create subclasses like `Car`, `Motorcycle`, and `Truck` that inherit from the `Vehicle` class. Each subclass can have its own specific properties and methods. For instance, the `Car` class can have a property called `numberOfDoors` and a method called `honk()`, while the `Motorcycle` class can have a property called `hasSidecar` and a method called `revEngine()`. By using inheritance, we can avoid code duplication and create a clear hierarchy of classes that represent different types of vehicles, while still maintaining the common functionality in the base `Vehicle` class. This allows for better organization, code reusability, and easier maintenance of the vehicle management system.

## best practices for inheritance in TypeScript
1. **Use Inheritance Only When Necessary**: Inheritance should be used when there is a clear "is-a" relationship between the parent and child classes. If the relationship is not clear, consider using composition instead, where classes are composed of other classes rather than inheriting from them.    
2. **Keep the Inheritance Hierarchy Simple**: Avoid creating deep inheritance hierarchies, as they can become difficult to manage and understand. Try to keep the hierarchy as flat as possible, and consider using interfaces or abstract classes to define common behavior without creating a complex inheritance structure.  


## industry usecases of inheritance
1. **UI Component Libraries**: Inheritance is commonly used in UI component libraries to create a base component class that defines common properties and methods for all components, such as rendering and event handling. Subclasses can then inherit from the base component class and add specific functionality for different types of components, such as buttons, input fields, and dropdowns. This allows for code reuse and consistency across the UI components, making it easier to maintain and extend the library.    
2. **Game Development**: Inheritance is widely used in game development to create a hierarchy of classes that represent different types of game objects. For example, you can have a base class called `GameObject` that contains common properties and methods for all game objects, such as position, velocity, and a method called `update()`. Then, you can create subclasses like `Player`, `Enemy`, and `NPC` that inherit from the `GameObject` class and add specific properties and methods for each type of game


## in backend development:
- Inheritance is commonly used in backend development to create a hierarchy of classes that represent different types of entities or models in an application. For example, in a blogging platform, you can have a base class called `Post` that contains common properties and methods for all types of posts, such as `title`, `content`, and a method called `publish()`. Then, you can create subclasses like `Article`, `Video`, and `Podcast` that inherit from the `Post` class and add specific properties and methods for each type of post, such as `author` for articles, `duration` for videos, and `host` for podcasts. This allows for code reuse and consistency across the different types of posts, making it easier to maintain and extend the backend application. Additionally, inheritance can be used to create a hierarchy of classes that represent different types of users in an application, such as `Admin`, `Editor`, and `Subscriber`, which can inherit from a base class called `User` that contains common properties and methods for all users, such as `username`, `email`, and a method called `login()`. This promotes code reuse and helps to maintain a clear structure for user management in the backend application.
